Data Structures / Arrays / Sequences

Arrays / Sequences

To rodzina struktur przechowujących elementy w pewnej kolejności. W praktyce większość codziennej pracy na danych zaczyna się właśnie tutaj.

Fixed-size array

Ogólna nazwa dla tablic o stałym rozmiarze, np. C-owych tablic i std::array. Dobre, gdy rozmiar znasz z góry.

  • należy do: Arrays / Sequences
  • mocna strona: przewidywalność i prostota

Dynamic array

Ogólna kategoria, do której należą m.in. C++ vector, C# List<T> i Java ArrayList. Ciągła pamięć i świetna lokalność cache.

  • należy do: Arrays / Sequences
  • typowo: szybki dostęp po indeksie i append

Deque

Double-ended queue z blokową pamięcią. Daje random access, ale nie jest linked listą.

  • należy do: Arrays / Sequences
  • typowo: gdy potrzebujesz także prepend

Doubly linked list

Dwukierunkowa linked list. W C++ odpowiada jej zwykle list, w C# LinkedList<T>.

  • należy do: Arrays / Sequences
  • uwaga: brak random access

Singly linked list

Jednokierunkowa linked list, np. C++ forward_list. Jeszcze prostsza i lżejsza niż wersja dwukierunkowa.

  • należy do: Arrays / Sequences
  • typowo: rzadko używana w CP

Data Structures / Hash-based

Hash-based

Ta rodzina stawia na szybkie wyszukiwanie po kluczu dzięki hashowaniu. W praktyce często chodzi o średni czas O(1).

Hash map

Ogólna nazwa dla struktur typu C++ unordered_map, C# Dictionary<TKey, TValue> i Python dict.

  • należy do: Hash-based
  • uwaga: w CP czasem dodaje się custom hash

Hash set

Zbiór oparty o hash table, np. C++ unordered_set, C# HashSet<T> i Python set.

  • należy do: Hash-based
  • typowo: membership test bez utrzymywania kolejności

Data Structures / Trees

Trees

Tu zaczynają się podrodziny. Nie każde drzewo służy do tego samego: jedne pilnują porządku kluczy, inne wspierają specyficzne operacje na przedziałach lub prefiksach.

Data Structures / Trees / BST family

BST family

To drzewa wyszukiwań binarnych. Utrzymują relację lewy mniejszy, prawy większy, a różne warianty balansują drzewo na różne sposoby.

Data Structures / Trees / BST family / RB-tree

RB-tree

Zbalansowane BST używane typowo pod map i set.

Data Structures / Trees / BST family / AVL

AVL tree

Mocniej pilnuje wysokości niż RB-tree, często kosztem większej liczby rotacji.

Data Structures / Trees / BST family / Treap

Treap

Łączy porządek BST z losowym priorytetem. Popularny w edukacyjnych implementacjach.

Data Structures / Trees / BST family / Splay

Splay tree

Przesuwa często używane elementy bliżej korzenia przez operację splay.

Data Structures / Trees / B-tree family

B-tree family

Rodzina drzew wielodrogowych, projektowana pod pracę z pamięcią masową i blokami dyskowymi. Klasyczna domena to bazy danych i systemy plików.

Data Structures / Trees / B-tree family / B-tree

B-tree

Przechowuje klucze i dzieci wewnątrz jednego węzła, minimalizując liczbę odwołań.

Data Structures / Trees / B-tree family / B+ tree

B+ tree

Wariant, w którym dane zwykle lądują w liściach, a liście są połączone sekwencyjnie.

Data Structures / Trees / CP trees

CP trees

To grupa drzew kojarzonych bardziej z algorytmiką i zadaniami niż z ogólnymi kontenerami standardowych bibliotek. Zwykle odpowiadają na konkretne typy zapytań.

Data Structures / Trees / CP trees / Segment tree

Segment tree

Drzewo do zapytań i aktualizacji na przedziałach.

Data Structures / Trees / CP trees / Fenwick tree

Fenwick tree

Lżejsza struktura do sum prefiksowych i wybranych aktualizacji.

Data Structures / Trees / CP trees / Trie

Trie

Drzewo prefiksowe przydatne zwłaszcza dla stringów i słowników.

Data Structures / Heaps

Heaps

Heapy są zbudowane pod szybki dostęp do minimum lub maksimum, a nie pod pełny porządek wszystkich elementów.

Priority queue

Abstrakcyjna kolejka priorytetowa. W C++ zwykle używasz priority_queue, w Javie PriorityQueue.

  • należy do: Heaps
  • typowo: pobieranie największego lub najmniejszego elementu

Data Structures / Graph-based

Graph-based

To bardziej rodzina reprezentacji niż jeden konkretny kontener standardowej biblioteki. Gdy problem opisuje wierzchołki i krawędzie, zwykle pracujesz na strukturach grafowych.

Adjacency list

Najczęstsza reprezentacja w CP, zwykle jako lista sąsiedztwa oparta o sekwencje lub jako lista krawędzi.

Adjacency matrix

Prostsza koncepcyjnie, ale pamięciowo droższa. Dobra raczej dla małych grafów.

Data Structures / Specialized

Specialized

Tu trafiają struktury projektowane pod bardzo konkretny typ operacji. Nie są ogólnymi kontenerami, ale potrafią rozwiązać konkretną klasę problemów bardzo elegancko.

Union-Find / DSU

Struktura do zarządzania rozłącznymi zbiorami i pytaniami o spójność komponentów.

  • należy do: Specialized
  • typowo: łączenie zbiorów i sprawdzanie, czy elementy są w tym samym zbiorze
