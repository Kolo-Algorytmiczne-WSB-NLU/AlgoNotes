Szczegółowa złożoność

O(n^2)

Quadratic time najczęściej bierze się z dwóch zagnieżdżonych pętli albo z brute force po wszystkich parach.

Wróć do przeglądu

Złożoności / O(n^2)

Najkrótsza intuicja

Każdy z każdym

Jeśli dla każdego elementu patrzysz jeszcze na wiele innych elementów, koszt zaczyna rosnąć kwadratowo. Dla małych danych to bywa w porządku, ale przy większych limitach szybko boli.

Typowe przykłady

  • dwie pełne pętle po tablicy
  • przejście po wszystkich polach macierzy
  • brute force po parach

Typowe limity

W praktyce często da się to obronić dla n ≤ 3000, a przy bardzo małych danych nawet niżej to zupełnie akceptowalny wybór.

Ważna uwaga

W Big-O patrzymy na najgorszy przypadek, więc pętla, która czasem robi mniej, nadal często liczy się jako O(n^2).

Snippet

Kod w 4 językach

Przykład: dwie pętle
#include <stdio.h>

int main(void) {
    int n = 4;
    int count = 0;

    for (int i = 0; i < n; i++) {
        for (int j = 0; j < n; j++) {
            count++;
        }
    }

    printf("%d\n", count);
    return 0;
}
Przykład: nested loops
#include <iostream>

int main() {
    int n = 4;
    int count = 0;

    for (int i = 0; i < n; ++i) {
        for (int j = 0; j < n; ++j) {
            ++count;
        }
    }

    std::cout << count << '\n';
    return 0;
}
Przykład: kwadratowa liczba iteracji
using System;

class Program
{
    static void Main()
    {
        int n = 4;
        int count = 0;

        for (int i = 0; i < n; i++)
        {
            for (int j = 0; j < n; j++)
            {
                count++;
            }
        }

        Console.WriteLine(count);
    }
}
Przykład: każda para indeksów
n = 4
count = 0

for i in range(n):
    for j in range(n):
        count += 1

print(count)
