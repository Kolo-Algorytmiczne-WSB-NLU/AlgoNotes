Szczegółowa złożoność

O(log n)

Logarytm rośnie bardzo wolno. Jeśli w każdej iteracji problem zmniejsza się mniej więcej o połowę, zwykle lądujesz właśnie tutaj.

Wróć do przeglądu

Złożoności / O(log n)

Najkrótsza intuicja

Bardzo powolny wzrost

log(1_000_000) to tylko około 20. W wywiadach i zadaniach O(log n) zwykle oznacza binary search, lookup w balanced BST albo przetwarzanie cyfr liczby.

Typowe przykłady

  • binary search i jego warianty
  • lookup w balanced BST
  • operacje na B-tree lub RB-tree

Domyślna baza

Jeśli nie zaznaczono inaczej, przyjmujemy zwykle log_2(n), czyli logarytm o podstawie 2.

Typowe limity

Ta klasa często pojawia się przy dużych ograniczeniach, np. dla n > 10^8.

Snippet

Kod w 4 językach

Przykład: dzielenie problemu na pół
#include <stdio.h>

int main(void) {
    int n = 100000000;
    int steps = 0;

    while (n > 0) {
        steps++;
        n /= 2;
    }

    printf("%d\n", steps);
    return 0;
}
Przykład: logarytmiczna pętla
#include <iostream>

int main() {
    int n = 100000000;
    int steps = 0;

    while (n > 0) {
        ++steps;
        n /= 2;
    }

    std::cout << steps << '\n';
    return 0;
}
Przykład: każda iteracja zmniejsza n
using System;

class Program
{
    static void Main()
    {
        int n = 100000000;
        int steps = 0;

        while (n > 0)
        {
            steps++;
            n /= 2;
        }

        Console.WriteLine(steps);
    }
}
Przykład: logarytmiczny spadek
n = 100_000_000
steps = 0

while n > 0:
    steps += 1
    n //= 2

print(steps)
