Szczegółowa złożoność

O(n)

Linear time zwykle oznacza przejście przez liniową strukturę stałą liczbę razy. To bardzo częsty i praktyczny koszt w CP.

Wróć do przeglądu

Złożoności / O(n)

Najkrótsza intuicja

Jedno przejście po danych

Najczęściej chodzi o pętlę po tablicy lub linked liście, dwa wskaźniki, część zachłannych rozwiązań, traversal drzewa lub grafu, a także klasyczne użycie stacka i queue.

Typowe przykłady

  • przejście po tablicy lub liście
  • two pointers
  • traversal drzewa albo grafu

Stałe i dodatki

W Big-O ignorujemy stałe i niższe rzędy, więc 5n + 17 nadal pozostaje w O(n).

Typowe limity

Liniowe rozwiązania są zwykle bezpieczne dla danych rzędu n ≤ 10^6.

Snippet

Kod w 4 językach

Przykład: pojedyncza pętla
#include <stdio.h>

int main(void) {
    int n = 8;
    int sum = 0;

    for (int i = 0; i < n; i++) {
        sum += i;
    }

    printf("%d\n", sum);
    return 0;
}
Przykład: liniowe sumowanie
#include <iostream>

int main() {
    int n = 8;
    int sum = 0;

    for (int i = 0; i < n; ++i) {
        sum += i;
    }

    std::cout << sum << '\n';
    return 0;
}
Przykład: jedno przejście
using System;

class Program
{
    static void Main()
    {
        int n = 8;
        int sum = 0;

        for (int i = 0; i < n; i++)
        {
            sum += i;
        }

        Console.WriteLine(sum);
    }
}
Przykład: liniowa pętla
n = 8
sum_value = 0

for i in range(n):
    sum_value += i

print(sum_value)
