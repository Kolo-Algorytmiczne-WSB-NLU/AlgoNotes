Szczegółowa złożoność

O(2^n)

To koszt wykładniczy. Rośnie bardzo szybko i zwykle oznacza backtracking, podzbiory albo rekurencję, która rozgałęzia się na dwa podobne podproblemy.

Wróć do przeglądu

Złożoności / O(2^n)

Najkrótsza intuicja

Dwa rozgałęzienia na krok

Taki koszt często pojawia się w kombinatoryce i backtrackingu. Bez memoizacji bardzo szybko staje się za duży, więc zwykle działa tylko dla małych wartości n.

Typowe przykłady

  • przegląd wszystkich podzbiorów
  • naiwny Fibonacci
  • część problemów backtrackingowych

Typowe limity

Najczęściej trzeba zostać przy n ≤ 20, chyba że zastosujesz memoizację albo inne przycięcie stanu.

Ważna uwaga

Ten koszt bywa trudniejszy do zauważenia na pierwszy rzut oka, bo często ukrywa się w rekurencji.

Snippet

Kod w 4 językach

Przykład: naiwny Fibonacci
#include <stdio.h>

int Fib(int n) {
    if (n == 0 || n == 1) {
        return 1;
    }
    return Fib(n - 1) + Fib(n - 2);
}

int main(void) {
    printf("%d\n", Fib(6));
    return 0;
}
Przykład: klasyczny Fibonacci jak na screenie
#include <iostream>

int Fib(int n) {

    }
    return Fib(n - 1) + Fib(n - 2);
}

int main() {
    std::cout << Fib(6) << '\n';
    return 0;

Przykład: rekurencja rozgałęzia się dwa razy
using System;

class Program
{
    static int Fib(int n)
    {
        if (n == 0 || n == 1)
        {
            return 1;
        }
        return Fib(n - 1) + Fib(n - 2);
    }

    static void Main()
    {
        Console.WriteLine(Fib(6));
    }
}
Przykład: wykładnicza rekurencja
def fib(n: int) -> int:
    if n == 0 or n == 1:
        return 1
    return fib(n - 1) + fib(n - 2)


print(fib(6))
