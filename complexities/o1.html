Szczegółowa złożoność

O(1)

Constant time complexity. Koszt jest stały, więc rozmiar danych nie zmienia liczby kroków potrzebnych dla jednej operacji.

Wróć do przeglądu

Złożoności / O(1)

Najkrótsza intuicja

Stały koszt

Najczęściej oznacza to hashmap lookup, array access lub update, push i pop na stosie albo podstawienie do wzoru matematycznego. W praktyce taki koszt bywa wygodny nawet przy bardzo dużych limitach wejścia.

Typowe przykłady

  • odczyt i zapis array[i]
  • średnie wyszukiwanie w hash map
  • push i pop na stacku

Praktyczny obraz

Jeśli wykonujesz tylko stałą liczbę prostych operacji, całe rozwiązanie nadal zostaje w O(1).

Typowe limity

Taki koszt jest zwykle bezpieczny nawet dla bardzo dużych danych, często rzędu n > 10^9, o ile nie powtarzasz operacji zbyt wiele razy.

Snippet

Kod w 4 językach

Przykład: stały dostęp do tablicy
#include <stdio.h>

int main(void) {
    int values[5] = {3, 8, 13, 21, 34};
    int answer = values[3];  /* O(1) access */
    values[1] = 99;          /* O(1) update */

    printf("%d\n", answer);
    return 0;
}
Przykład: stały dostęp do dynamic array w C++
#include <iostream>
#include <vector>

int main() {
    std::vector<int> values = {3, 8, 13, 21, 34};
    int answer = values[3];  // O(1) access
    values[1] = 99;          // O(1) update

    std::cout << answer << '\n';
    return 0;
}
Przykład: tablica w C#
using System;

class Program
{
    static void Main()
    {
        int[] values = { 3, 8, 13, 21, 34 };
        int answer = values[3]; // O(1)
        values[1] = 99;         // O(1)

        Console.WriteLine(answer);
    }
}
